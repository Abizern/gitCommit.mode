gitcommit.mode by Abizern

A SubEthaEdit mode that adds some syntax colouring to git commit messages.

Nothing too special, just a slightly easier way of adding a bit of syntax colouring to git commit messages by using SubEthaEdit as the editor on Mac OS X. The colouring isn't carried over into the log.

Inspired by this post doing something similar with Mercurial

Dependencies

You need to have git installed and SubEthaEdit.

Install

SubEthaEdit

Download the gitCommit.mode to your local machine. Double clicking on it will give you the option of installing it for the current user or for all users.

.gitconfig

Add the following line to the .gitconfig file [core] section.

[core] 
   editor = /usr/bin/see -w -r -o new-window -j 'Git editor' -m gitCommit -g 1:0

You might have to edit this if your see tool isn't in the same location. Now, whenever you do a git commit without a -m inline commit, it will open SubEthaEdit for you to enter the comment.
